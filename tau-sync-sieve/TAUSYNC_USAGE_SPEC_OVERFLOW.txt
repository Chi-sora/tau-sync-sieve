TAU SYNC WINDOWS BINARY INDEX

1. Environment
- Windows 11
- MinGW-w64
- Command Prompt
- build.bat
- Multithreaded execution

2. Build

    build.bat

3. Index generation

    tausync.exe make-index <MAX> index.tsb

Example:

    tausync.exe make-index 1000000 index.tsb --threads 8 --flush-mib 1024

The index always covers the range:

    1 ... MAX

4. Label lookup

    tausync.exe label index.tsb 109

Output examples:

    109 prime
    101112 composite
    unknown

"unknown" means that the value could not be classified by the TAU SYNC index.

5. Twin prime search

    tausync.exe twin-top10 index.tsb

The command searches only the previously generated index.tsb.
No additional mathematical search is performed.

6. Progress display

Long-running operations display progress approximately every 5 seconds.

7. Memory management

Rows are generated in blocks.
Output is accumulated in a write buffer and flushed periodically.

Example:

    --flush-mib 1024

This reduces the number of disk writes while avoiding excessive RAM usage.

8. Overflow policy

- All counters and indices use 64-bit integer types.
- Intermediate multiplications should be guarded against overflow.
- Build range must satisfy the representable range of uint64_t.
- Buffer sizes are checked before allocation.

9. TAU SYNC ONLY POLICY

This implementation is intended to operate as TAU SYNC ONLY.

During index generation, label lookup, and twin-prime lookup, the implementation does NOT invoke:

- isprime
- Miller-Rabin
- APR-CL
- ECPP
- external omega calculators
- external factorization engines

Classification is performed only from TAU SYNC data.

If a value cannot be classified by TAU SYNC, it is returned as:

    unknown

10. File layout

All source files and this document are intended to reside directly in the repository root.
No additional source subdirectories are required.
